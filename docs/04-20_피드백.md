# 캡스톤 디자인 피드백 정리
**날짜:** 2026-04-20  
**출처:** 실제 피드백 회의(음성 녹취) + 정리 문서 상호 대조

---

## 1. 발표 방식 및 구성

### 지적 사항
- **기술 설명 부족**: "이렇게 해서 이렇게 된다"는 결과만 보여주고 로직·구현 방식 설명이 없음
  - 예: 몸무게 30kg 입력 → 체중으로 출력되는 **산출 로직**을 설명해야 함
- **시나리오 발표 전환 필요**: 현재 기능 나열 방식 → 사용자 흐름 기반 시나리오 방식으로 변경
  - 예: 가입 → 정보 입력 → 식사 추가 → 식단 분석까지 이어지는 흐름으로 발표
- **발표 연습 부족**: 최종 발표 전 충분한 연습 필요

### 권장 사항
- 파워포인트 슬라이드에 **알고리즘 흐름도** 포함
- 데이터가 어디서 왔는지(식약처 등) 시각적으로 명시
- 특수 케이스(알레르기 있는 사용자가 된장찌개 선택하는 경우 등) 시나리오 준비

## 2. 주요 기능별 피드백

### 2-1. 사용자 등록 (온보딩)
| 구분 | 내용 |
|------|------|
| 현재 상태 | 버튼/폼 방식 → **채팅 방식으로 전환 완료** |
| 요구 사항 | LLM(Gemma 계열)으로 대화에서 정보 추출 → DB 저장 로직 구현 |
| 추가 요구 | ChatGPT/LLM API 연동하여 이름 등 정보를 자유 대화에서 추출하는 방식 설명 필요 |
| 비고 | 기본 정보(키·몸무게 등)는 폼으로, 기호·알레르기 등 추가 정보만 채팅으로 수집하는 **혼합 방식** 고려 권장 |

### 2-2. 식단 분석 & 리포트
| 구분 | 내용 |
|------|------|
| 현재 상태 | 일간 리포트 화면 구성됨 |
| 요구 사항 | **일간 / 주간 / 월간** 3단계로 나누어 구현 |
| 데이터 | 식약처 약 18,000개 식품 성분 데이터 활용 (출처 명시 필요) |
| 비고 | 아침·점심·저녁 항목별로 분리 표시, 자동 영양 분석 흐름 설명 |

### 2-3. 개인 맞춤 추천 알고리즘 ⭐ (가장 중요)
| 구분 | 내용 |
|------|------|
| 현재 상태 | RAG(ChromaDB + KR-SBERT + LLM) 기반 추천 구현됨 |
| 요구 사항 | **추천 카테고리 세분화** 필요 |
| 카테고리 예시 | - 다이어트를 위한 추천<br>- 사용자 기호에 따른 추천<br>- 질환 맞춤 추천 (당뇨, 고혈압 등) |
| 핵심 시나리오 | 볶음밥 먹은 후 다이어트 식단 추천 → 이 흐름의 알고리즘 설명 필요 |
| 특수 조건 처리 | "튀긴 음식 싫다" 같은 음식 조건 입력 → 식약처 데이터에서 필터링하는 로직 필요 |

### 2-4. 알레르기 정보 관리
| 구분 | 내용 |
|------|------|
| 현재 상태 | DB에 알레르기 필드 존재, LLM이 음식명에서 추론 |
| 지적 사항 | 된장찌개에 **땅콩이 들어가는지** 실제 성분 데이터 기반으로 필터링해야 함 |
| 요구 사항 | 식약처 데이터에서 성분 정보 추출 → 알레르기 유발 메뉴 자동 제외 기능 |
| 해결 방향 | 식약처 18,000개 데이터에 성분 포함 여부 확인 가능한지 검토 필요 |

## 3. 향후 과제 (우선순위순)

| 우선순위 | 항목 | 설명 |
|----------|------|------|
| P0 | 추천 알고리즘 카테고리 세분화 | 다이어트/기호/질환별 탭 구성 |
| P0 | 발표 시나리오 작성 | 사용자 흐름 기반 슬라이드 |
| P1 | 주간/월간 리포트 화면 구현 | 현재 일간만 있음 |
| P1 | 알레르기 성분 데이터 연동 | 식약처 원재료 데이터 확인 |
| P2 | 기술 문서화 | 알고리즘 흐름도, 데이터 출처 정리 |
| P2 | 외부 로그인 연동 | 네이버/카카오 OAuth (권장 사항) |
| P3 | 온보딩 혼합 방식 개선 | 폼 + 채팅 최적 조합 |

---

## 4. 현재 구현 완료 항목 (긍정 피드백)

- 구현량이 많다는 칭찬 ("구현을 많이 했다")
- 채팅 방식 온보딩 전환 완료
- LLM 기반 정보 추출 및 DB 등록 로직 구현
- RAG 기반 맞춤 추천 기능 동작
- 식약처 데이터 활용한 영양 분석

## 5. 음성 녹취 vs 정리 문서 비교

| 항목 | 음성 녹취에서 추가된 내용 | 정리 문서에서 추가된 내용 |
|------|--------------------------|--------------------------|
| 알레르기 | 된장찌개 + 땅콩 구체적 예시 언급 | 알레르기 테이블 DB 추가 방식 |
| 추천 | 볶음밥 먹은 후 시나리오 구체 언급 | 추천 카테고리 세분화 방향 |
| 발표 | 연습 부족 직접 지적 | 시나리오 발표 방식 전환 권고 |
| 데이터 | 식약처 18,000개 언급 | 외부 데이터 출처 명시 필요 |
| 로그인 | 네이버/카카오 연동 언급 | 외부 서비스 연동 권장 |
| 온보딩 | 기본 폼 + 추가만 채팅으로 혼합 제안 | 채팅 방식 전환 + LLM 추출 로직 |
